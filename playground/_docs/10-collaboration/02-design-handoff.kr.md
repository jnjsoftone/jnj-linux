# 디자인 핸드오프 가이드

**목적**: 디자인에서 개발로의 원활한 전환 보장

이 문서는 디자인팀에서 개발팀으로 디자인을 핸드오프하는 프로세스와 체크리스트를 설명합니다.

## 🎯 핸드오프 목표

- **명확성**: 개발자가 디자인 의도를 이해
- **완전성**: 모든 필요한 에셋과 명세 제공
- **일관성**: 시스템 가이드라인과 디자인 정렬
- **실현 가능성**: 기술적으로 구현 가능한 디자인

## 🔄 핸드오프 프로세스

### 1단계: 디자인 준비 (디자이너)

**일정**: 핸드오프 미팅 전

#### 디자인 명세 완성
- [ ] 모든 화면이 모든 중단점용으로 디자인됨 (모바일, 태블릿, 데스크톱)
- [ ] 인터랙티브 상태 문서화 (hover, active, disabled, error, loading)
- [ ] 애니메이션 및 전환 지정 (지속 시간, 이징, 트리거)
- [ ] 반응형 동작 정의
- [ ] 접근성 요구사항 명시 (ARIA 레이블, 키보드 탐색)

#### 디자인 파일 정리
- [ ] 파일 이름 규칙 따름
- [ ] 아트보드/프레임 적절히 레이블링
- [ ] 디자인 시스템의 컴포넌트 사용
- [ ] 디자인 토큰 적용 (색상, 타이포그래피, 간격)
- [ ] 변형 및 상태 명확히 표시

#### 에셋 준비
- [ ] SVG 형식으로 아이콘 내보내기
- [ ] 이미지 최적화 (WebP, 적절한 해상도)
- [ ] 일러스트레이션을 벡터 형식으로
- [ ] 폰트 포함 또는 참조
- [ ] 에셋 이름 규칙 따름

### 2단계: 핸드오프 미팅 (디자이너 + 개발자)

**소요 시간**: 기능당 30-60분

#### 안건
1. **디자인 워크스루** (15분)
   - 디자이너가 디자인 발표
   - 사용자 플로우 및 인터랙션 설명
   - 주요 디자인 결정 강조

2. **기술 논의** (15분)
   - 개발자가 명확한 질문
   - 기술적 제약사항 논의
   - 잠재적 과제 식별
   - 구현 접근 방식 합의

3. **에셋 검토** (10분)
   - 모든 에셋 사용 가능 확인
   - 에셋 형식 및 품질 확인
   - 이름 규칙 확인

4. **다음 단계** (10분)
   - 개발 일정 설정
   - 디자인 검토 체크포인트 일정
   - 작업 항목 할당
   - 커뮤니케이션 채널 정의

### 3단계: 개발 (개발자)

**일정**: 구현 중

#### 개발자 책임
- [ ] 모든 디자인 명세 검토
- [ ] 지정된 채널에서 질문
- [ ] 명세에 따라 구현
- [ ] 디자인과의 편차 플래그
- [ ] 마일스톤에서 디자인 검토 요청

### 4단계: 디자인 검토 (디자이너 + 개발자)

**일정**: 구현 마일스톤에서 (25%, 50%, 75%, 100%)

#### 검토 체크리스트
- [ ] 시각적 정확성 (색상, 타이포그래피, 간격)
- [ ] 인터랙티브 동작 (hover, click, focus 상태)
- [ ] 중단점 간 반응형 동작
- [ ] 애니메이션 및 전환
- [ ] 접근성 기능
- [ ] 엣지 케이스 및 오류 상태

```markdown
2. 사용자 액션 → 시스템 응답

## 화면

### 화면 1: [화면 이름]

**목적**: 이 화면이 달성하는 것

**Figma 링크**: [디자인 URL]

#### 레이아웃
- 컨테이너 너비: [값]
- 패딩: [값]
- 그리드: [열 x 간격]

#### 컴포넌트

#### 상태
- 기본: [설명]
- 로딩: [설명]
- 성공: [설명]
- 오류: [설명]

#### 반응형 동작
- **모바일 (< 768px)**: [변경사항]
- **태블릿 (768-1024px)**: [변경사항]
- **데스크톱 (> 1024px)**: [변경사항]

#### 접근성
- ARIA 레이블: [세부사항]
- 키보드 탐색: [탭 순서]
- 스크린 리더 텍스트: [내용]
```

## 🛠️ 도구 및 리소스

### 개발자 도구
- **Storybook**: 컴포넌트 개발
- **Browser DevTools**: 구현 확인

### 협업 도구

  - `#design-dev` 채널
- **Jira/Linear**: 작업 추적

## ✅ 핸드오프 체크리스트

### 디자이너 핸드오프 전 체크리스트
- [ ] 모든 화면 디자인됨 (모바일, 태블릿, 데스크톱)
- [ ] 모든 상태 문서화됨
- [ ] 애니메이션 지정됨
- [ ] 에셋 내보내기 및 정리됨
- [ ] 명세 문서 생성됨
- [ ] 디자인 시스템 준수 확인됨
- [ ] 접근성 요구사항 명시됨
- [ ] 핸드오프 미팅 예약됨

### 개발자 핸드오프 후 체크리스트
- [ ] 디자인 파일 검토됨
- [ ] 명세 이해됨
- [ ] 에셋 다운로드됨
- [ ] 기술적 질문 해결됨
- [ ] 구현 접근 방식 합의됨
- [ ] 일정 추정됨
- [ ] 첫 마일스톤 예약됨

### 완료 체크리스트
- [ ] 모든 화면 구현됨
- [ ] 모든 상태 기능함

- [ ] 모든 중단점에서 반응형
